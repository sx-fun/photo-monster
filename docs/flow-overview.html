⚡ 性能优化效果

❌ 优化前（问题点）

1000万像素图直方图 遍历全部像素
计算耗时（大图） ~2–5 秒卡顿
CSS 动画周期 1s / 2s（偏短）
滚动监听 app.js 重复注册
图片数据提取 同步阻塞主线程
Canvas 读取 未启用优化标志

✅ 优化后（改进点）

1000万像素图直方图 每16px取1个
计算量削减 减少 93.75%
CSS 动画周期 1.5s / 3s（流畅）
滚动监听 rAF 单一版本
图片数据提取 requestIdleCallback
Canvas 读取 willReadFrequently: true
📊 直方图动态采样策略（计算量削减比例）
超大图（＞1000万像素，如 A7R V 6100万像素） 削减 93.75%
大图（500–1000万像素） 削减 87.5%
中图（200–500万像素） 削减 75%
小图（＜200万像素） 削减 50%
🔧 四项核心优化技术
① 动态采样
根据像素总量自动调整 step（2/4/8/16），保证视觉准确性的同时大幅减少计算量
② CSS will-change
提前告知浏览器哪些元素会变化，GPU 提前分配图层，避免动画触发重排
③ requestAnimationFrame
滚动事件节流，每帧最多执行一次，移除 app.js 重复注册的监听器
④ requestIdleCallback
图片 EXIF 提取推迟到浏览器空闲期，不阻塞渲染和用户交互
