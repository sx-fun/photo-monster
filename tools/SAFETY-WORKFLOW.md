# Photo Monster 安全修改工作流

## 概述

本文档定义了 Photo Monster 网站所有修改必须遵循的安全流程，确保：
- 不引入安全漏洞
- 不破坏原有功能
- 可追溯、可回滚
- 避免重复处理相同问题

---

## 工作流程图

```
┌─────────────────────────────────────────────────────────────┐
│  1. START - 开始修改                                         │
│     命令: node workflow.js start "修改说明"                   │
│     动作: 自动创建完整备份                                    │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  2. EDIT - 进行修改                                          │
│     动作: 使用编辑器修改代码                                  │
│     注意: 随时可回滚                                          │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  3. CHECK - 安全检测                                         │
│     命令: node workflow.js finish                           │
│     动作: 自动运行安全检测脚本                                │
└─────────────────────────────────────────────────────────────┘
                              ↓
                    ┌─────────────────┐
                    │  检测是否通过?   │
                    └─────────────────┘
                      ↓ 是          ↓ 否
            ┌──────────┐      ┌──────────┐
            │ 4a. PASS │      │ 4b. FAIL │
            │ 手动验证  │      │ 修复问题  │
            │ 部署更新  │      │ 重新检测  │
            │          │      │ 或回滚   │
            └──────────┘      └──────────┘
```

## 检测项目清单

### 自动检测（脚本执行）

| 检测项 | 说明 | 失败处理 |
|--------|------|---------|
| 敏感信息扫描 | 检查API Key、密码等硬编码 | **禁止部署**，必须移除 |
| XSS漏洞检查 | 检查innerHTML、eval等危险操作 | **禁止部署**，必须修复 |
| 代码规范 | console.log数量、未使用变量等 | 警告，建议修复 |
| 路径安全 | 检查路径遍历风险 | **禁止部署**，必须修复 |
| HTML规范 | charset、viewport等必要标签 | 警告，建议修复 |

### 手动验证（人工检查）

| 检查项 | 验证方法 | 通过标准 |
|--------|---------|---------|
| 照片上传功能 | 上传测试图片 | 正常显示、无错误 |
| EXIF读取功能 | 上传带EXIF的照片 | 参数正确显示 |
| AI分析功能 | 点击分析按钮 | 返回有效结果 |
| LocalStorage存储 | 刷新页面 | 历史记录保留 |
| 响应式布局 | 调整窗口大小 | 各尺寸正常显示 |
| 导航链接 | 点击所有导航 | 跳转正确 |
| 跨页面一致性 | 浏览各页面 | 风格统一 |

## 命令参考

### 主要命令

```bash
# 开始修改（必须）
node workflow.js start "添加直方图功能"

# 完成修改并检测（必须）
node workflow.js finish

# 查看当前状态
node workflow.js status

# 回滚到修改前
node workflow.js rollback

# 查看完整指南
node workflow.js guide
```

### 辅助命令

```bash
# 单独运行安全检测
node safety-check.js
node safety-check.js js/photo-analysis.js

# 创建备份
node pre-edit-backup.js "备份说明"

# 列出所有备份
node pre-edit-backup.js list

# 恢复到指定备份
node pre-edit-backup.js restore backup-2026-03-28T09-30-00
```

## 规则与约束

### 必须遵守

1. **每次修改前必须运行 `start`**
   - 禁止直接修改不备份

2. **每次修改后必须运行 `finish`**
   - 禁止跳过检测直接部署

3. **检测不通过禁止部署**
   - ERROR级别问题必须修复
   - WARNING级别问题建议修复

4. **保留所有检测报告**
   - 用于问题追溯
   - 定期归档（保留最近20份）

### 建议遵守

1. **小步快跑**
   - 每次修改聚焦一个功能
   - 便于定位和回滚

2. **及时提交**
   - 完成一个功能就 finish
   - 不要堆积大量修改

3. **定期清理**
   - 每月清理旧备份（保留最近10个）
   - 每季度归档旧报告

## 常见问题

### Q: 忘记运行 start 直接修改了怎么办？

**A**: 立即停止，手动创建备份：
```bash
node pre-edit-backup.js "紧急备份-添加直方图"
# 然后继续正常流程
node workflow.js finish
```

### Q: 检测失败但想先部署再修复？

**A**: **绝对禁止**。检测失败意味着存在安全漏洞或功能缺陷，必须先修复。

### Q: 如何查看历史修改记录？

**A**: 
```bash
node workflow.js status
# 或查看文件
tools/.workflow-state.json
```

### Q: 回滚后如何重新修改？

**A**: 回滚后状态重置为 idle，直接运行 start 开始新的修改流程。

### Q: 多个文件如何检测？

**A**: 
```bash
# 检测指定文件
node safety-check.js js/file1.js js/file2.js

# 检测所有文件
node safety-check.js
```
